# Практики 11–12 — RBAC-панель для недвижимости

Учебный full-stack проект по дисциплине «Разработка фронтенда и бэкенда», 4-й семестр, 2025/2026 учебный год.

## Описание проекта

Приложение предоставляет единый интерфейс для трёх ролей пользователей:

- **Покупатель (`user`)** — просматривает каталог объектов недвижимости.
- **Риелтор (`seller`)** — создаёт и редактирует объявления.
- **Администратор (`admin`)** — управляет всеми объявлениями и пользователями системы: меняет роли, блокирует аккаунты.

Данные хранятся в памяти (in-memory), персистентность между перезапусками не предусмотрена — проект является учебным стендом.

## Быстрый старт

### Требования

- Node.js 18+
- npm 9+

### 1. Запуск бэкенда

```bash
cd 11-12/backend
npm install
npm start
```

Сервер запустится на `http://localhost:3002`.

### 2. Запуск фронтенда

```bash
cd 11-12/frontend
npm install
npm start
```

Клиент откроется на `http://localhost:3003`.

> Оба сервиса должны работать одновременно. Открывайте приложение по адресу **http://localhost:3003**.

## API-эндпоинты

Базовый URL: `http://localhost:3002/api`

### Здоровье сервиса

| Метод | Путь      | Описание            | Авторизация |
|-------|-----------|---------------------|-------------|
| GET   | `/health` | Проверка работы API | Нет         |

### Аутентификация

| Метод | Путь              | Описание                          | Авторизация |
|-------|-------------------|-----------------------------------|-------------|
| POST  | `/auth/register`  | Регистрация нового пользователя   | Нет         |
| POST  | `/auth/login`     | Вход, возвращает access + refresh | Нет         |
| POST  | `/auth/refresh`   | Обновление access-токена          | Нет         |
| GET   | `/auth/me`        | Данные текущего пользователя      | Bearer JWT  |

#### Тело запроса `POST /auth/register`

```json
{
  "username": "newuser",
  "password": "pass1234",
  "role": "user"
}
```

Допустимые роли: `user`, `seller`, `admin`. Пароль — минимум 4 символа.

#### Тело запроса `POST /auth/login`

```json
{
  "username": "buyer",
  "password": "buyer123"
}
```

#### Ответ `POST /auth/login`

```json
{
  "accessToken": "<jwt>",
  "refreshToken": "<jwt>",
  "user": {
    "id": "abc12345",
    "username": "buyer",
    "role": "user",
    "isBlocked": false,
    "createdAt": "2026-04-08T12:00:00.000Z"
  }
}
```

### Пользователи (только `admin`)

| Метод  | Путь         | Описание                                    |
|--------|--------------|---------------------------------------------|
| GET    | `/users`     | Список всех пользователей                   |
| GET    | `/users/:id` | Данные конкретного пользователя             |
| PUT    | `/users/:id` | Обновление логина, роли, статуса блокировки |
| DELETE | `/users/:id` | Блокировка пользователя                     |

#### Тело запроса `PUT /users/:id`

```json
{
  "username": "newname",
  "role": "seller",
  "isBlocked": false
}
```

> Администратор не может заблокировать самого себя.

### Недвижимость

| Метод  | Путь               | Роли                | Описание               |
|--------|--------------------|---------------------|------------------------|
| GET    | `/properties`      | user, seller, admin | Список всех объявлений |
| POST   | `/properties`      | seller, admin       | Создать объявление     |
| GET    | `/properties/:id`  | user, seller, admin | Получить объект по ID  |
| PUT    | `/properties/:id`  | seller, admin       | Обновить объявление    |
| DELETE | `/properties/:id`  | admin               | Удалить объявление     |

#### Тело запроса `POST /properties` / `PUT /properties/:id`

```json
{
  "title": "Квартира в центре",
  "propertyType": "Квартира",
  "address": "Москва, ул. Тверская, 1",
  "description": "Светлая квартира с видом на город",
  "price": 15000000,
  "area": 54
}
```

Все поля обязательны. `price` и `area` — положительные числа.

## Архитектура аутентификации

```
Клиент                                  Сервер
  │                                        │
  ├─── POST /auth/login ──────────────────►│
  │◄── { accessToken, refreshToken } ──────┤
  │    (сохраняются в localStorage)        │
  │                                        │
  ├─── GET /properties  [Bearer access] ──►│
  │◄── 200 OK ─────────────────────────────┤
  │                                        │
  │    (access-токен истёк через 15 мин)   │
  ├─── GET /auth/me  [Bearer expired] ────►│
  │◄── 401 ────────────────────────────────┤
  │                                        │
  │    (axios interceptor автоматически)   │
  ├─── POST /auth/refresh [refreshToken] ─►│
  │◄── { accessToken, refreshToken } ──────┤
  │    (токены обновлены в localStorage)   │
  │                                        │
  ├─── GET /auth/me  [Bearer new] ────────►│
  │◄── 200 OK ─────────────────────────────┤
```

- **Access-токен** живёт **15 минут**, передаётся в заголовке `Authorization: Bearer <token>`.
- **Refresh-токен** живёт **7 дней**, хранится на сервере в `Map` и в `localStorage` клиента.
- При блокировке пользователя все его refresh-токены немедленно отзываются.
- Если обновление токена не удалось — фронтенд генерирует событие `auth:expired`, интерфейс сбрасывается в состояние гостя.

## Клиент (Frontend)

### Ключевые модули

**[`api/client.js`](frontend/src/api/client.js)** — Axios-клиент:
- Автоматически подставляет `Authorization: Bearer` из `localStorage`.
- Response interceptor перехватывает `401`, вызывает `POST /auth/refresh` и повторяет исходный запрос с новым токеном.
- При невозможности обновить токен диспатчит событие `auth:expired` для глобальной обработки.

**[`App.js`](frontend/src/App.js)** — главный компонент:
- При загрузке проверяет наличие токена и восстанавливает сессию через `GET /auth/me`.
- Вычисляет права (`canViewProperties`, `canEditProperties`, `canManageUsers`, `canDeleteProperties`) на основе роли пользователя.
- Единый интерфейс адаптируется: неавторизованным пользователям показывается только форма входа.
